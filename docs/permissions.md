# 权限组说明

本文档描述 Mutsumi 当前实现中的 `Admin`、`User`、`Guest` 三个权限组。权限以服务端路由实际鉴权逻辑为准。

## 权限概览

| 功能 | Admin | User | Guest |
| --- | :---: | :---: | :---: |
| 登录并获取访问令牌 | ✓ | ✓ | ✓ |
| 查看自己的账号信息 | ✓ | ✓ | ✓ |
| 查看番剧列表和详情 | ✓ | ✓ | ✓ |
| 创建番剧和剧集记录 | ✓ | ✓ | ✓ |
| 更新自己的观看进度 | ✓ | ✓ | ✓ |
| 播放服务器视频 | ✓ | ✓ | ✓ |
| 查看 Mutsumi 下载任务 | ✓ | ✓ | ✓ |
| 添加和暂停下载任务 | ✓ | ✓ | — |
| 读取种子元数据和文件列表 | ✓ | ✓ | ✓ |
| 删除番剧、下载任务和文件 | ✓ | — | — |
| 查看和修改 qBittorrent 配置 | ✓ | — | — |
| 查看全部用户 | ✓ | — | — |
| 创建、编辑和删除用户 | ✓ | — | — |

## Admin

`Admin` 拥有全部普通使用能力和系统管理能力。

管理权限包括：

- 查看全部用户及其权限组。
- 创建 `Admin`、`User` 或 `Guest` 用户。
- 修改任意用户的用户名、密码和权限组。
- 删除用户。
- 读取和修改 qBittorrent 地址、账号、下载目录及分享率限制。
- 删除番剧数据库记录、关联观看进度、qBittorrent 任务和已下载文件。

当前限制：

- 系统尚未阻止删除最后一个 Admin。
- 系统尚未阻止 Admin 删除或降级自己的账号。
- 修改当前账号用户名后，已有访问令牌可能失效，需要重新登录。

## User

`User` 是标准使用权限组，可使用番剧播放和下载相关功能。

具体权限包括：

- 查看全局番剧库。
- 创建番剧和剧集记录。
- 查看和更新自己的观看进度。
- 播放服务器上的视频文件。
- 查看 `Mutsumi` 分类下的 qBittorrent 任务。
- 添加下载任务、暂停下载任务。
- 查询种子元数据和种子文件列表。

`User` 无法管理用户、修改 qBittorrent 服务配置或删除番剧及其文件。

## Guest

`Guest` 是一个可登录的数据库用户权限组，不等同于未登录访客。

Guest 当前可以：

- 查看和创建番剧记录。
- 保存自己的观看进度并播放视频。
- 查看下载任务。
- 查询种子元数据和文件列表。

Guest 不允许：

- 添加或下载番剧任务。
- 暂停下载任务。

Guest 同样无法执行 Admin 管理操作。

## 匿名访问

未登录请求方不属于 `Guest` 权限组。匿名访问仅限：

- 查询服务健康状态和初始化状态。
- 在服务尚未初始化时创建首个管理员。
- 使用用户名和密码登录。
- 查询可用权限组枚举。

其他番剧、视频、下载和设置接口均要求有效的 Bearer Token。

## 后续建议

- 继续明确 `User` 与 `Guest` 在创建番剧记录等操作上的差异。
- 禁止删除或降级最后一个 Admin。
- 根据当前用户权限隐藏客户端中的管理菜单，同时保留服务端权限校验。
- 为高风险管理操作增加审计日志。
